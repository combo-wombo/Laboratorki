Таблица "Законодатели":
Идентификатор законодателя (Primary Key)
Имя законодателя
Адрес законодателя
Домашний телефон законодателя
Служебный телефон законодателя

Таблица "Комиссии":
Идентификатор комиссии (Primary Key)
Название комиссии
Профиль комиссии (образование, медицина, жилье и т.д.)
Идентификатор председателя (Foreign Key, ссылается на Законодатели)

Таблица "Члены_Комиссий":
Идентификатор члена (Primary Key)
Идентификатор законодателя (Foreign Key, ссылается на Законодатели)
Идентификатор комиссии (Foreign Key, ссылается на Комиссии)

Таблица "Заседания":
Идентификатор заседания (Primary Key)
Дата и время заседания
Идентификатор комиссии (Foreign Key, ссылается на Комиссии)
Члены, принявшие участие (может содержать список имен членов)

Проблемы, из-за которых эта структура не находится в 1НФ, 2НФ и 3НФ одновременно:

Таблица "Заседания" нарушает 1НФ, так как поле "Члены, принявшие участие" может содержать множество значений.
Для нормализации таблицы "Заседания" и устранения нарушения 1НФ, вы можете создать дополнительную таблицу для хранения информации о членах, принявших участие в заседаниях.
CREATE TABLE Участники_Заседаний (
    Идентификатор INT PRIMARY KEY,
    Идентификатор_заседания INT,
    Имя VARCHAR(255),
    FOREIGN KEY (Идентификатор_заседания) REFERENCES Заседания(Идентификатор)
);
Теперь вы можете изменить таблицу "Заседания", чтобы удалить поле "Члены, принявшие участие". Таблица "Заседания" должна выглядеть так:
sql
Copy code
CREATE TABLE Заседания (
    Идентификатор INT PRIMARY KEY,
    Дата_и_время DATETIME
);

Таблица "Члены_Комиссий" нарушает 2НФ, так как один член может быть членом нескольких комиссий, и идентификатор члена зависит от комбинации (Идентификатор законодателя, Идентификатор комиссии).
Для нормализации таблицы "Члены_Комиссий" и устранения нарушения 2НФ, вы можете разделить эту таблицу на две отдельные таблицы. Одна из них будет отвечать за информацию о членах комиссии, а другая будет связывать законодателей и комиссии, в которых они участвуют. 
CREATE TABLE Члены_Комиссии (
    Идентификатор_члена INT PRIMARY KEY,
    Идентификатор_законодателя INT,
    FOREIGN KEY (Идентификатор_законодателя) REFERENCES Законодатели(Идентификатор)
);
CREATE TABLE Участники_Комиссий (
    Идентификатор INT PRIMARY KEY,
    Идентификатор_члена INT,
    Идентификатор_комиссии INT,
    FOREIGN KEY (Идентификатор_члена) REFERENCES Члены_Комиссии(Идентификатор_члена),
    FOREIGN KEY (Идентификатор_комиссии) REFERENCES Комиссии(Идентификатор)
);

Таблица "Комиссии" нарушает 3НФ, так как поле "Профиль комиссии" зависит только от Идентификатора комиссии и не имеет отношения к другим атрибутам.
Для нормализации таблицы "Комиссии" и устранения нарушения 3НФ, вы можете создать отдельную таблицу для хранения профилей комиссий и установить связь между таблицами по идентификатору комиссии. 
CREATE TABLE Профили_Комиссий (
    Идентификатор_профиля INT PRIMARY KEY,
    Идентификатор_комиссии INT,
    Профиль VARCHAR(255),
    FOREIGN KEY (Идентификатор_комиссии) REFERENCES Комиссии(Идентификатор)
);
CREATE TABLE Комиссии (
    Идентификатор INT PRIMARY KEY,
    Название VARCHAR(255),
    Идентификатор_председателя INT,
    FOREIGN KEY (Идентификатор_председателя) REFERENCES Законодатели(Идентификатор)
);
